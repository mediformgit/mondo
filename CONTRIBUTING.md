# 貢献ガイド — CONTRIBUTING

問道に関わってくれてありがとうございます。
**まず [PRINCIPLES.md](./PRINCIPLES.md)（綱領）を読んでください。** すべての貢献は、その思想の上に成り立ちます。

問道のいちばんの財産は、**思想と質の一貫性**です。だから貢献は歓迎しつつ、**採否は運営者（メンテナ）が最終判断**します。これは貢献者を信頼しないからではなく、サイトの芯を守るためです。

---

## 貢献の種類と、いちばん簡単な始め方

ほとんどの貢献は**コードを書かずに、データを足すだけ**で完結します。

| やりたいこと | 編集する場所 |
|---|---|
| 講義を追加・改善 | [`lib/lessons.ts`](./lib/lessons.ts) |
| 型（パターン）を追加 | [`lib/patterns.ts`](./lib/patterns.ts) |
| 道場の例を追加（`demoReply` も） | [`lib/presets.ts`](./lib/presets.ts) |
| 誤字・事実誤り・リンク切れの修正 | 該当ファイル |
| 翻訳・アクセシビリティ・デザイン改善 | 相談の上で |

## 進め方

1. Issue で提案・相談（大きな変更は特に。小さな修正はいきなりPRでもOK）。
2. ブランチを切る：`git checkout -b lesson/add-xxx`
3. 編集する。
4. **`npm run build` がローカルで通ることを確認**（型・lintエラーを残さない）。
5. Pull Request を作成 → CI と Vercel プレビューで確認 → レビュー → merge。

## ✅ 思想チェック（PRに必ず通す）

提出前に、[PRINCIPLES.md §IV](./PRINCIPLES.md) の判断テストを自分で確認してください。要点：

- [ ] **入口をせばめていない**（初心者を締め出さない）
- [ ] **無料・無登録・広告なし・追跡なし**を守っている
- [ ] **ユーザーの鍵・データを手元から奪わない**（CSPの `connect-src` を広げない）
- [ ] **呪文ではなく原理**を教えている
- [ ] **陳腐でない**（量産AIデザイン・安易な実装でない）
- [ ] **文系・理系に偏っていない**
- [ ] 内容が**正確**（AIの例は幻覚を疑い、事実は裏取り済み）

ひとつでも満たさないPRは、修正をお願いするか、見送ります。

## コンテンツの質の基準

- **正確さ第一。** 生成AIの挙動・数値・固有名詞は変わりやすい。断定しすぎず、必要なら一次情報の確認を促す書き方に。
- **声を保つ。** 編集的で、品位があり、説教くさくない。テンプレ的な“AI記事”にしない。
- **転移する原理を。** 「これを貼れば最強」ではなく、「なぜ効くか」を教える。

## ライセンス（貢献＝同意）

PRを送ることで、あなたは自分の貢献を次のライセンスで提供することに同意します：

- **コード** … MIT（[LICENSE](./LICENSE)）
- **コンテンツ本文** … CC BY-SA 4.0（[LICENSE-CONTENT.md](./LICENSE-CONTENT.md)）

他者の著作物（文章・画像・コード）を**無断で持ち込まない**でください。引用は出典を明記し、ライセンスを尊重すること。

## 行動規範

[CODE_OF_CONDUCT.md](./CODE_OF_CONDUCT.md) に従ってください。要は、**学ぶ人すべてに開かれた場**を、互いに敬意をもって保つことです。
