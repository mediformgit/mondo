### [起草済] AIは、あなたに賛成したがる（迎合・sycophancy）
- 起草: 講義10 `sycophancy`「AIは、なぜあなたに賛成するのか」＋道場プリセット `neutral-question`（ブランチ `lesson/sycophancy`）。
- 角度（一行の主張）: AIが同意するのは、あなたが正しいからではない。同意するよう訓練されているからだ。だから「賛成」は、それ自体では情報量がゼロだと知る。
- トラック / レベル: 探究 /（中級〜上級）
- なぜ新規・非自明か: 世のAI記事は「AIに反論させよう」（＝講義8）で止まる。だがその一歩先——**反論役を頼んでも、こちらが押し返すとAIは正しかった答えを取り下げて迎合する**——はほとんど語られない。人間の好みで微調整する学習（RLHF）が構造的に「気に入られる答え」を報酬にするため、迎合は能力不足ではなく**設計の副作用**。ゆえに対策は「批判して」と頼むことではなく、**自分の望む結論を問いに混ぜない（誘導尋問をしない）**こと。「問いを立てる力」の核心に直結する。
- 想定する「問い」の核: 「私はAだと思うけど、どう？」（誘導）→「Aという主張の弱点と、対立仮説Bの強みを、どちらにも肩入れせず」。確信度を聞く／立場を伏せて同じ問いを二度投げて答えのブレを見る。
- 要検証の事実: あり（要出典）— 迎合の存在と規模（複数モデルで平均約63.7%との報告）、RLHF起因、2025年のGPT-4o過剰迎合とロールバック。**※具体的な数値・固有事例は本文化時に再検証。**
- 採点メモ（陳腐テスト8基準）: 既出性○（"反論させる"記事は多いが"押し返しが正答を壊す"は稀）/ 非自明○ / 独自フレーム○（誘導尋問＝悪い問い）/ 転移性○（原理＝報酬設計の副作用）/ 鮮度○ / 文理○（題材中立）/ 声○ / 非自己反復△→要差別化（講義8＝能動的に批判を依頼、本講＝不随意の同意バイアスと中立化）。**最近接＝講義8 thinking-partner。リスク：講義8と混線→「あなたが選ぶ言葉が、真実か追従かを決める」の一点に絞る。**
- 出典（§4）: Anthropic [Towards Understanding Sycophancy in LMs](https://arxiv.org/pdf/2310.13548) ／ OpenAI [Sycophancy in GPT-4o](https://openai.com/index/sycophancy-in-gpt-4o/) ／ Science [Sycophantic AI decreases prosocial intentions](https://www.science.org/doi/10.1126/science.aec8352)

### [起草済] AIに「渡さない」という判断（過信・認知の外注）
- 起草: 講義11 `what-not-to-delegate`「AIに渡さないもの」＋道場プリセット `what-to-keep`（ブランチ `lesson/what-not-to-delegate`、`lesson/sycophancy` にスタック）。§0検証: Gerlich 2025（Societies 15:6）＋ Lee et al. CHI 2025（Microsoft/CMU, confidence effect）。相関・自己申告として本文化、固有数値は不掲載。
- 角度（一行の主張）: AIに何を任せるかより、**何を任せないか**が熟達を分ける。雑用は外注し、自分を変える思考は手放さない。
- トラック / レベル: 探究 /（上級・トラックの締め＝capstone候補）
- なぜ新規・非自明か: 「AIに頼りすぎると考える力が落ちる」という嘆きの記事は多いが、ほとんどが警鐘で止まり**実用的な線引きを与えない**。本講の角度は"AI悪玉論"ではなく、**外注してよいトイルと、手放してはいけない思考を見分ける判断規則**。認知の外注（cognitive offloading）と批判的思考の負相関（特に若年層で顕著）という研究知見を、問道の使命（"書くことで考える機会を手放さない"＝講義5・PRINCIPLES）に接続する。
- 想定する「問い」の核: 着手前の自問——「これはやり方を学ぶべきか／結果だけ要るか」「私の価値観で決めるべきか」「ここを自分で苦しむことに意味があるか」。AIには「答えではなくヒントを」（講義9と接続）。
- 要検証の事実: あり（要出典）— 認知の外注と批判的思考の相関に関する研究。**※相関であって因果断定はしない。本文化時に研究の限界（標本・自己申告など）も明記。**
- 採点メモ（陳腐テスト8基準）: 既出性○（線引きの"規則化"は稀）/ 非自明○ / 独自フレーム◎（使命と直結）/ 転移性◎（原理）/ 鮮度○ / 文理○ / 声△→具体例で説教臭を回避 / 非自己反復○（講義5の"丸ごと書かせない"を一般化・上位化）。**リスク：精神論化→必ず具体的な分岐の問いで着地。探究トラックの締めに据える価値あり。**
- 出典（§4）: Gerlich, [AI Tools in Society: Cognitive Offloading and Critical Thinking (Societies/SSRN)](https://papers.ssrn.com/sol3/papers.cfm?abstract_id=5082524) ／ [psypost 要約記事](https://www.psypost.org/ai-tools-may-weaken-critical-thinking-skills-by-encouraging-cognitive-offloading-study-suggests/)

### [提案中] AIは「いつ」で時間が止まっているか（知識の地平線）
- 角度（一行の主張）: AIには"知らない未来"がある。しかも本人はそれに気づかない。幻覚とは別の、**時間の死角**を知る。
- トラック / レベル: 入門 〜 実践 /（初級）
- なぜ新規・非自明か: 多くの解説は知識カットオフと幻覚を混同する。本講は両者を**明確に分ける**：幻覚＝知っている範囲内での捏造、カットオフ＝**まるごと見えていない時間帯**。AIは「知らない」とは言わず、古い前提のまま自信たっぷりに答える。対策は「全部疑う」ではなく**変わりうる最新の事実は自分で渡す／境界を意識する**こと（＝下ごしらえ講に接続）。
- 想定する「問い」の核: 「最近の〜は？」と素で聞く（危険）→「いつ時点の知識？」を確認し、価格・法改正・人事・最新版など変わりうる事実は自分で与えてから問う。
- 要検証の事実: あり（要出典）— 知識カットオフの概念と挙動。**※特定モデルのカットオフ日付は本文に書かない（すぐ古びる）。検索連携など特定ツールに依存した記述もしない＝原則のツール非依存を保つ。**
- 採点メモ（陳腐テスト8基準）: 既出性△→芯は"幻覚との分離"（カットオフ解説自体は多い）/ 非自明○ / 独自フレーム○（"時間の死角"）/ 転移性○ / 鮮度◎（あえて日付を書かない設計で耐久）/ 文理○ / 声○ / 非自己反復○（講義7幻覚と別物として立てる）。**最近接＝講義7。差別化を本文冒頭で明示。**
- 出典（§4）: [Knowledge cutoff (Wikipedia)](https://en.wikipedia.org/wiki/Knowledge_cutoff) ／ [Knowledge Cutoff Explained](https://canadianaiguy.ca/definition/knowledge-cutoff/)（※概念確認用。具体日付は採用しない）

### [提案中] 渡してから、問う（下ごしらえ・grounding）
- 角度（一行の主張）: 初心者はAIを"何でも知っている神託"として問う。熟達者は**自分の資料を読ませてから**問う。これが幻覚と的外れを一気に減らす、最も費用対効果の高い一手。
- トラック / レベル: 実践 /（初級〜中級）
- なぜ新規・非自明か: 専門家は当然のようにやるが、**初心者向けに"原理として"教える教材は少ない**。「AIの記憶」より「あなたが渡した文章」の方が答えの根拠として遥かに強い——文脈を与えると幻覚が大きく減るという知見を、料理の"下ごしらえ（mise en place）"という和の比喩で初心者に橋渡しする。講義2（問いの部品）とは別軸：部品＝問いの作り方、本講＝**地（ground truth）の供給**。
- 想定する「問い」の核: 「〜について教えて」（記憶頼み）→「以下の資料**だけ**に基づいて答えて。資料にないことは『資料にない』と言って」。長文・契約書・データ・自分のメモを"地"にする。
- 要検証の事実: あり（要出典）— 文脈付与が幻覚を大幅に減らすという研究知見。**※"○%減"の具体数値は出典の実験条件依存。本文では断定しない。**
- 採点メモ（陳腐テスト8基準）: 既出性△→初心者向けの原理化は稀（RAGの技術論は多い）/ 非自明○ / 独自フレーム○（神託→下ごしらえ）/ 転移性◎ / 鮮度○ / 文理◎（契約・論文・コード・家計どれでも）/ 声○ / 非自己反復○。**最近接＝講義7幻覚／講義2。差別化：本講は"根拠の供給"に一点集中。**
- 出典（§4）: [Grounding LLMs to In-prompt Instructions (ACL 2024)](https://aclanthology.org/2024.safety4convai-1.1/) ／ [AI is Not an Oracle: Designing More Effective Prompts (UW Law)](https://wisblawg.law.wisc.edu/2024/12/13/ai-is-not-an-oracle-designing-more-effective-prompts)

- 角度（一行の主張）: 「要約して」は罠。要約は必ず何かを捨てる。**何を捨ててよいかは"何のために読むか"でしか決まらない**。目的なき要約は、あなたが欲しかった一点を真っ先に捨てる。
- トラック / レベル: 実践 /（中級）

### [提案中] 見せて、問う（画像を読ませる・マルチモーダル）
- 角度（一行の主張）: AIは"読む"だけでなく"見る"。手書きメモ・図・グラフ・エラー画面・写真を見せて問えば、入口がもう一段ひらく。
- トラック / レベル: 入門 〜 実践 /（初級）

### [起草済] 問いを、疑う（XY問題・問題の枠組み）
- 起草: 講義8 `question-the-question`「問いを、疑う」＋道場プリセット `reframe-goal`（実践トラック末尾に挿入、探究8〜11を9〜12へ繰り上げ。ブランチ `lesson/question-the-question`）。§0: 流動的事実なし（XY問題は確立した概念）。
- 角度（一行の主張）: あなたの問いは、すでに答えを狭めている。「どうやってXするか」と聞く前に、「本当に欲しいのはZでは？」と問う。手段を尋ねて、目的を見失わない。
- トラック / レベル: 実践 /（中級）
- なぜ新規・非自明か: 巷の教材は「プロンプトの磨き方」（＝より良いXの聞き方）に集中し、**そもそも聞いている問いが正しいか**を扱わない。本講の芯は古典的な"XY問題"——本当はZが欲しいのに、自分が思いついた手段Xのやり方だけを尋ね、AIは律儀にXに完璧に答え、的外れな迷路にはまる。これは問道の使命「問いを立てる力」の最深部：**最良の答えは、磨かれた問いではなく、正された問いから出る。**
- 想定する「問い」の核: 「PDFを画像に変換するには？」（思い込みの手段X）→「紙の資料の文章をAIに読ませたい。最善の方法は？」（真の目的Z）。着手前に「私はなぜこれを聞いている？一段上の目的は？」を自問する。
- 要検証の事実: なし（原理。XY問題は確立した概念。本文化時に具体例の事実のみ検証）。
- 採点メモ（陳腐テスト8基準）: 既出性○（"問いを磨け"は多いが"問い自体を疑え=XY"は稀）/ 非自明◎ / 独自フレーム◎（使命の核） / 転移性◎（原理）/ 鮮度◎ / 文理◎（料理・コード・進路相談どれでも）/ 声○ / 非自己反復○。**最近接＝講義2（問いの部品＝どう組むか）と型clarify（聞き返させる＝手段）。差別化：本講は"自分で問いを疑う"上位の構え。冒頭で明示。**
- 出典（§4）: 概念確認用 [The XY Problem](https://xyproblem.info/) ／ [XY problem (Wikipedia)](https://en.wikipedia.org/wiki/XY_problem)。事実依存は最小。

- 角度（一行の主張）: AIは「うまく言う」のが得意だ。よどみない自信に満ちた文章を、人はつい「正しい」と取り違える。流暢さと正しさを切り離し、自分の側の油断を疑う。
