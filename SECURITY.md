# セキュリティポリシー — SECURITY

## 報告のしかた

脆弱性を見つけた場合は、**公開のIssueにはせず**、非公開で運営者に報告してください
（GitHub の「Security advisories（Report a vulnerability）」、またはリポジトリ記載の連絡先）。
誠実に対応し、可能な範囲で**報告者をクレジット**します。報奨金はありません。

対応の目安：受領の確認は数日以内、修正は深刻度に応じて。

## 特に重視する領域（このサイトの信頼の核）

問道は **BYOK（利用者が自分のAPIキーを持ち込む）** モデルです。次に該当する報告を最優先で扱います：

- **ユーザーのAPIキー／プロンプト本文が、`api.anthropic.com` 以外へ送られうる経路。**
  （CSPの `connect-src` を回避できる、キーがログ・解析・第三者に渡る、など）
- CSP・各種セキュリティヘッダの回避。
- なりすまし・フィッシングを助ける挙動。
- XSS など、上記につながりうるクライアント側の脆弱性。

> 設計上の不変条件：CSPの `connect-src` は `self` と `https://api.anthropic.com` のみ
> （[next.config.mjs](./next.config.mjs)）。これを崩す変更・回避は、原則違反として扱います（[PRINCIPLES.md](./PRINCIPLES.md)）。

## 対象範囲

- ✅ このリポジトリのコードと、公式にデプロイされたサイト。
- ❌ 非公式なフォーク／クローン（[BRAND.md](./BRAND.md) 参照）。利用者は公式ドメイン以外でAPIキーを入力しないでください。
- ❌ Anthropic 側のAPI・インフラ（Anthropic へ直接報告を）。

報告ありがとうございます。利用者の安全は、このプロジェクトの根幹です。
