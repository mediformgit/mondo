# /propose-themes — 講義テーマをゼロから起案する

問道の新しい講義テーマを、思いつきでなく**研究と分析**から起案し、順位付けして
[docs/THEME-BACKLOG.md](../../../docs/THEME-BACKLOG.md) に追記する。**推奨モデル: Opus 4.8**（新規性・価値の判断と研究統合のため）。

## 最初に必ず読む
1. [PRINCIPLES.md](../../../PRINCIPLES.md) — 不可侵の原則。テーマはこれに適合すること。
2. [docs/QUALITY-RUBRIC.md](../../../docs/QUALITY-RUBRIC.md) — 陳腐化ガードレール。テーマ段でも適用する。
3. [lib/lessons.ts](../../../lib/lessons.ts) と [lib/patterns.ts](../../../lib/patterns.ts) — 既存の講義・型（重複回避・ギャップ把握）。
4. [docs/THEME-BACKLOG.md](../../../docs/THEME-BACKLOG.md) — 既存の台帳（重複回避）。

## 手順

### 1. 候補を出す（6つの源泉）
- **使命からの逆算**：「問いを立てる力を、初心者→本質へ」を技能の階段と捉え、まだ無い“段”を探す。
- **ギャップ分析**：既存9講・型を読み、抜け／文理の偏り／初級↔上級の段差を特定。
- **Web研究（差別化のため）**：`web_search` で「世に既にある教え方」を調べ、**わざと見落とされている角度**を取りにいく。新しいモデル能力・よくある誤解・つまずきの定番も拾う。
- **第一原理／LLMの挙動**：LLMが実際どう振る舞うか、なぜその問いが効くかの**非自明だが普遍的な洞察**を翻訳。
- **失敗モード**：幻覚・曖昧指示・鵜呑み・反復しない等を「問い→講義」に変換。
- **解析シグナル（あれば）**：修了率・道場の使われ方など需要データ。

### 2. 絞り込む（採点 → 均衡）
各候補を **新規性・洞察の価値・耐久性（原理か流行か）・使命適合・被覆（穴を埋めるか）・検証可能性** で採点。
既存講義・台帳と**重複排除**し、トラック/文理/レベルの**ポートフォリオ均衡**を取る。
QUALITY-RUBRIC の陳腐テストで自己採点し、× が残るものは落とす。

### 3. 出力
- 上位 **5〜8件**を、台帳のフォーマット（状態 `提案中`）で docs/THEME-BACKLOG.md に**追記**する（既存を消さない）。
- 各テーマに、ルーブリック §4 の添付（**なぜ陳腐でないか／要検証の事実／参照した出典**）を必ず含める。
- 新規性の高い順に並べる。

### 4. 報告
追記した件数と、特に推す上位2〜3件を一言で運営者に伝える。**自動で本文化はしない**（それは `/create-lesson`）。

## 原則
- テーマは PRINCIPLES.md に適合（無料/入口/文理/原理）。流行追いの使い捨てネタは出さない。
- **新規性は“奇をてらう”ことではない。** 実用と初心者適合（入口をせばめない）を満たした上での新規性であること。難解さ・マニアックさを新規性と取り違えない。
- 既存と重複しない。台帳は追記のみ（既存項目を消さない）。
- このスキルはテーマを**提案するだけ**。採否の最終判断は運営者（または自動進行設定）。
